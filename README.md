# EdiTask Web

![代替テキスト](./image/editask_icon.png)

EdiTask は、タスクを素早く編集できることを目的にしたタスク管理ツールです。

EdiTask は、1行1タスクのテキストを直接編集し、`Ctrl+S` でソートを行うことで、タスクの追加・整理・完了をできるだけ軽くできるようにしています。

このアプリは Tritask の考え方に強く影響を受けています。
参考: [Tritask のコンセプト](https://tritask.github.io/tritask-web/concept.html)

## 基本思想

- 動作が軽く、タスクの追加・編集が簡単である
- すべてのタスクを時間軸に入れる
- 今日と明日以降のタスクを分けて、今日のタスクは今日のうちにやり切る
- GTD のように雑多なタスクを集めても、プロジェクト単位に絞りこめる
- 保存時の並び替えは辞書順で、何度保存しても同じ結果になる

## 最初に貼るテンプレート

まずは `main` に以下をコピーして`Ctrl+S` して使うことをおすすめします。

```text
"""
この場所には自由なメモを書くことができます
"""
n -- TODAY TODO hold:0
n 1行１タスクです。Ctrl+Qでスタート、完了できます
日報を書く rep:1  m:30m
燃えるゴミ出し rep:1 days:月木
-- TODAY END hold:1
-- WEEK END hold:土
n+4 チームミーティング rep:7 m:1h30m
n+100 AirTag電池交換 rep:365
n+200 インフルワクチン予約を行う rep:340 days:1
00:00 00:00 2024/01/01 月 -- YESTERDAY DONE
14:00 15:32 2026/06/24 水 チームミーティング rep:7 m:1h30m
```

## 基本操作

| 操作              | 内容                                  |
| ----------------- | ------------------------------------- |
| `Ctrl+S`          | 保存と正規化とソート                  |
| `Ctrl+Q`          | 現在行の 開始/完了 を切り替え         |
| `Ctrl+Shift+F`    | Enhanced Filter を開閉                |
| `Ctrl+F/H`        | 通常検索/置換                         |
| `Ctrl+R`          | 現在行の `ref:` / `r:` を開く         |
| `Ctrl+Shift+Up`   | 選択行または現在行の日付を1日前へ移動 |
| `Ctrl+Shift+Down` | 選択行または現在行の日付を1日後へ移動 |

`Ctrl+Q` は次の順で切り替わります。

```text
開始時刻が記録 -> 完了時刻が記録 -> 両方クリア -> 開始時刻が記録 -> ...
```

`rep` があるタスクを完了すると、次回分のタスクが自動生成されます。

## タスクの書き方

基本は 1 行 1 タスクです。

```text
            2026/06/26 金 レポートを書く
09:10       2026/06/26 金 作業中のタスク
09:10 10:20 2026/06/26 金 完了したタスク
```

開始/完了 欄は固定幅です。手で揃えなくても、保存時に整形されます。

## 日付入力

タスクは日付から始めるのが基本です。

```text
2026/06/26 タスク
```

短縮入力も使えます。

```text
n 買い物
n+2 週末までに確認
3/14 予約する
6/26+1 明日にずらしたタスク
```

- `n` は今日
- `n+2` は今日から2日後
- `M/D` または `MM/DD` は、今日以降で最も近いその日付
- `YYYY/MM/DD+1` は指定日から1日後

何も指定がないと、近い日付が補完されます。
日付を後から動かす場合は、`Ctrl+Shift+Up/Down` が便利です。
複数行を選択していれば、選択中のタスク日付をまとめて動かせます。

## ソート

`Ctrl+S` で保存すると、タスクはソートされます。

- 今日のタスク
- 未来のタスク
- 過去の完了済みタスク
- 日付のない行

同じ日付の中では辞書順で並びます。
順番を固定したい場合は、行頭に `01`, `02`, `10` のような番号を付けると安定します。

```text
            2026/06/26 金 01 レポートを書く
            2026/06/26 金 02 ミーティング準備
            2026/06/26 金 10 バックアップ確認
```

## Enhanced Filter

`Ctrl+Shift+F` で Enhanced Filter を開きます。

通常検索とは違い、指定した文字を含む行だけに表示を絞ります。
一度すべてのタスクを時系列に放り込み、
必要なときだけプロジェクト名やキーワードで絞り込む使い方に向いています。

```text
            2026/06/26 金 projectA 仕様を書く
            2026/06/26 金 projectA レビューする
            2026/06/27 土 projectB 請求書を確認
```

`projectA` で Enhanced Filter すると、projectA のタスクだけを見ながら編集できます。

`--` 行はFilter内容にかかわらず区切りとして表示されます。

## ルーチンワーク

`rep:n` は n 日後に次回タスクを生成します。

```text
            2026/06/26 金 日報を書く rep:1
            2026/07/03 金 チームミーティング rep:7
            2026/10/06 火 AirTag電池交換 rep:365
```

`days:` を組み合わせると、次回日付を指定曜日や指定日に寄せられます。

```text
            2026/06/26 金 燃えるゴミ出し rep:1 days:月木
            2026/11/01 日 インフルワクチン予約を行う rep:340 days:1
```

- `days:月` は月曜日まで先送りします
- `days:月水金` のように複数曜日も指定できます
- `days:1` は次の月の1日に移動します

## hold

`hold` は保存時に日付を移動するための属性です。

```text
            2026/06/26 金 -- TODO hold:0
            2026/06/27 土 -- TODAY END hold:1
            2026/06/27 土 -- WEEK END hold:土
```

- `hold:0` は常に今日
- `hold:1` は常に明日
- `hold:土` は次の土曜日

区切り行に付けると、今日や週末のブロックを自動的に維持できます。

## 見積もり

`m:` は作業見積もりです。
下部バーの `作業予測` に、今日の未完了タスクの見積合計が表示されます。

```text
            2026/06/26 金 メール返信 m:10m
            2026/06/26 金 設計を書く m:2h
            2026/06/26 金 レビューする m:2h10m
```

有効な例:

```text
m:10m
m:70m
m:2h
m:2h10m
m:1h70m
```

無効な形式は赤く表示され、作業予測に加算されません。

## ref

`ref:` または `r:` で別ファイルや URL を開けます。

```text
            2026/06/26 金 仕様を確認する r:Aの仕様
            2026/06/26 金 参考ページを見る ref:https://example.com
```

`Ctrl+R` で開きます。

- URL は新しいタブで開きます
- `r:Aの仕様` は `Aの仕様` ファイルを新しいタブで開きます
- まだ存在しないファイルを開くと、新規作成されます

`default` ファイルを作っておくと、新しく作られるファイルはその内容をコピーして始まります。プロジェクト用の初期レイアウトを決めたい場合は、`default` にテンプレートを書いておくと便利です。

## メモブロック

`"""` で囲んだ範囲は自由なメモとして扱えます。

```text
"""
会議メモ
リンクや補足を書いておく ref:meeting
"""
```

メモブロックは保存時のソート対象外です。Enhanced Filter の検索対象にもなりません。ただし `ref:` などのハイライトはメモ内でも表示されます。

## 表示

- `--` を含む区切り行は上罫線で表示されます
- 無効な属性値は赤く表示されます

## 保存とデータ

保存は `Ctrl+S` したときだけ行います。
保存時に正規化とソートも同時に行います。

Google アカウントでログインすると、自分のデータだけを参照できます。複数端末から同じタスクを開けます。

必要に応じて `Export` からテキストとして書き出せます。
